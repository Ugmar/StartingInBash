# StartingInBash

[cite_start]Репозиторий посвящен изучению написания скриптов на автоматизированном командном интерпретаторе **Bash**[cite: 30]. [cite_start]Основная цель проекта — разработка специализированных скриптов-компараторов для парсинга, фильтрации и сравнения различных типов данных (чисел и строк) в текстовых файлах, а также автоматизация их тестирования[cite: 30, 43, 53, 56].

## 🛠 Технологический стек

* [cite_start]**Командная оболочка:** Bash (`/bin/bash`) 
* [cite_start]**Утилиты фильтрации и поиска:** `grep` (с поддержкой расширенных регулярных выражений `ERE` через флаг `-E`, а также извлечения совпадений с `-o`) [cite: 43, 48, 49, 53, 56]
* [cite_start]**Потоковая обработка текста:** `cat`, `tail`, встроенные циклы и механизмы подстановки строк (`${variable%pattern}`) [cite: 47, 48, 49, 53]

## 📂 Архитектура проекта и модули

[cite_start]Проект разделен на 4 независимых подзадачи (`task1` — `task4`)[cite: 30, 32, 35, 38]. Каждая папка содержит:
1. [cite_start]`comparatorX` — исполняемый скрипт-компаратор, принимающий два файла для сравнения и опциональный флаг отладки.
2. [cite_start]`test` — автоматизированный тестовый движок, прогоняющий тесты из локальной директории[cite: 44, 50, 54, 57].
3. [cite_start]`tests/` — набор позитивных (`_eq`) и негативных (`_ne`) тестовых сценариев[cite: 30, 31, 32, 33, 34].

### 🔹 task1: Сравнение целых чисел
* [cite_start]**Что делает:** Выделяет из текстовых файлов все изолированные целые числа (как положительные, так и отрицательные) и сравнивает их последовательности, полностью игнорируя строки и вещественные числа[cite: 48, 49, 51].
* [cite_start]**Регулярное выражение / Логика:** Проверяет каждое отдельное слово на соответствие паттерну `^[+-]?[0-9]+$`[cite: 48, 49]. [cite_start]Также корректно обрабатывает и очищает символы возврата каретки (`\r`) для совместимости с Windows-форматом файлов[cite: 48, 49].

### 🔹 task2: Сравнение строк по маркеру
* [cite_start]**Что делает:** Ищет в файлах строку-маркер, начинающуюся с `string:`[cite: 53]. [cite_start]Компаратор проверяет на идентичность саму строку с маркером, а также весь текст, который идет строго *после* нее (`tail`), игнорируя любые различия в метаданных или строках до этого маркера[cite: 53, 55].
* [cite_start]**Особенности:** Чувствителен к пробельным символам и знакам табуляции после маркера[cite: 35, 53].

### 🔹 task3: Сравнение простых вещественных чисел
* [cite_start]**Что делает:** Извлекает из файлов все вещественные числа в стандартном формате с плавающей точкой (вида `1.2`, `-10.325`) и сравнивает их последовательности в текстовом виде[cite: 43, 45, 46]. [cite_start]Строки и целые числа без точки при анализе отбрасываются[cite: 43, 45].
* [cite_start]**Регулярное выражение:** Использует `[+-]?[0-9]+\.[0-9]+` для точечной выборки совпадений[cite: 43].

### 🔹 task4: Сравнение чисел в научной (экспоненциальной) нотации
* [cite_start]**Что делает:** Выделяет и сравнивает вещественные числа, которые могут быть записаны как в обычном формате с точкой, так и в научном формате с экспонентой (например, `153.38e2`, `-21.12e-5`, `12.1e+2`)[cite: 56, 57].
* [cite_start]**Регулярное выражение:** Использует паттерн `[+-]?[0-9]+\.[0-9]+([Ee][+-]?[0-9]+)?` для захвата экспоненциальной части[cite: 56].
